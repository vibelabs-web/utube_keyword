# Architecture Decision Records (ADR)

## ADR-001: SQLite 선택
- **날짜**: 2024-01-17
- **상태**: 승인됨
- **맥락**: 데이터베이스 선택
- **결정**: SQLite 사용
- **이유**: 단일 사용자, 설정 간편, 파일 기반으로 배포 용이
- **결과**: 캐시 저장에 충분, 필요 시 PostgreSQL 전환 가능

## ADR-002: 인증 없음
- **날짜**: 2024-01-17
- **상태**: 승인됨
- **맥락**: 인증 시스템 필요 여부
- **결정**: 인증 미구현
- **이유**: 로컬에서 본인만 사용하는 도구
- **결과**: 개발 속도 향상, 복잡도 감소

## ADR-003: React + FastAPI 조합
- **날짜**: 2024-01-17
- **상태**: 승인됨
- **맥락**: 프레임워크 선택
- **결정**: React 19 + FastAPI
- **이유**: 빠른 개발, Python의 텍스트 분석 라이브러리 활용
- **결과**: 풍부한 생태계, 자동 API 문서화
