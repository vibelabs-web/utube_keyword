# PRD (제품 요구사항 정의서)

> 유튜브 키워드 분석기 & 댓글 분석기 - 바이브코딩 채널 성장 가속화 도구

---

## MVP 캡슐

| # | 항목 | 내용 |
|---|------|------|
| 1 | 목표 | 바이브코딩 채널의 성장 속도를 높이기 위해 잘 될 콘텐츠 주제를 데이터 기반으로 찾는다 |
| 2 | 페르소나 | 바이브코딩 유튜버 (구독자 5,000명, 영상 50개, 성장 가속화 희망) |

## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

바이브코딩 채널을 운영하고 있지만 성장 속도가 답답합니다.
마치 안개 속에서 운전하는 것처럼, 어떤 주제가 시청자들에게 먹힐지 감으로만 판단하고 있습니다.

기존 도구(vidIQ, TubeBuddy)는 비용이 부담되고, 내가 원하는 정보만 빠르게 확인하기 어렵습니다.

### 1.2 목표 상황 (To-Be)

- 키워드를 검색하면 검색량과 경쟁도를 한눈에 파악
- 인기 영상의 댓글을 분석해서 시청자가 원하는 것을 파악
- 데이터에 기반한 콘텐츠 전략으로 채널 성장 가속화

### 1.3 왜 지금인가? (Why Now)

- AI/바이브코딩 트렌드가 급성장 중
- 빠르게 콘텐츠를 내야 경쟁에서 앞설 수 있음
- 직접 만들면 비용 없이 원하는 기능만 구현 가능

## 2. 사용자 페르소나

### 2.1 주요 페르소나

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "성장갈증 유튜버" |
| 나이/직업 | 바이브코딩 유튜버 |
| 핵심 니즈 | 채널 성장 속도를 높이고 싶음 |
| 불편 포인트 | 어떤 주제가 될지 모름, 기존 도구가 비쌈 |
| 사용 상황 | 컴퓨터에서 영상 기획 전에 웹사이트로 분석 |

### 2.2 페르소나의 하루 (시나리오)

1. 새로운 영상 주제를 고민하다가 도구 접속
2. "Claude 4.5" 키워드의 검색량과 경쟁도 확인
3. 관련 키워드 추천을 보고 더 좋은 주제 발견
4. 해당 주제의 인기 영상 댓글을 분석
5. 시청자들이 궁금해하는 점을 파악
6. 데이터에 기반한 영상 기획 시작

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (온보딩)
- [ ] 사용자로서, 별도 로그인 없이 바로 사용할 수 있어야 한다. 왜냐하면 나 혼자 쓸 도구이기 때문이다.

### FEAT-1: 키워드 분석 (MVP)
- [ ] 사용자로서, 키워드를 입력하면 검색량을 볼 수 있어야 한다. 왜냐하면 수요가 있는 주제를 찾아야 하기 때문이다.
- [ ] 사용자로서, 키워드의 경쟁 정도를 볼 수 있어야 한다. 왜냐하면 경쟁이 적은 블루오션을 찾아야 하기 때문이다.
- [ ] 사용자로서, 관련 키워드 추천을 받을 수 있어야 한다. 왜냐하면 더 좋은 주제를 발견할 수 있기 때문이다.

### FEAT-2: 댓글 분석 (MVP)
- [ ] 사용자로서, 영상 URL을 입력하면 댓글을 수집할 수 있어야 한다. 왜냐하면 시청자 반응을 알아야 하기 때문이다.
- [ ] 사용자로서, 댓글에서 자주 나오는 단어/주제를 볼 수 있어야 한다. 왜냐하면 관심사를 파악해야 하기 때문이다.
- [ ] 사용자로서, 시청자 요청사항을 추출할 수 있어야 한다. 왜냐하면 다음 콘텐츠 아이디어를 얻어야 하기 때문이다.

### FEAT-3: 분석 결과 저장 (v2 이후)
- [ ] 사용자로서, 분석 결과를 저장하고 비교할 수 있어야 한다.

## 7. 해결 가설과 실험

### 7.1 가설
> 만약 키워드 검색량과 경쟁도를 데이터로 확인하면, 더 좋은 콘텐츠 주제를 선정할 수 있을 것이다. 왜냐하면 감이 아닌 데이터에 기반한 의사결정이 가능하기 때문이다.

### 7.2 실험 루프

```
실험 → 관측 → 학습 → 다음 가설
```

| 단계 | 내용 |
|------|------|
| 실험 | 도구로 선정한 주제 vs 감으로 선정한 주제 비교 |
| 관측 | 각 영상의 24시간/1주일 조회수 비교 |
| 학습 | 도구 기반 주제가 20% 이상 높으면 성공 |
| 다음 | 분석 기준 정교화 또는 새로운 지표 추가 |
